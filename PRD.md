 # PRD
 제품명: 나만의 청음 기호 + 오선지 기반 청음 메모지 웹
 
 ---
 
 ## 1. 제품 개요
 
 이 제품은 노래 청음 및 보컬 연습을 위한 웹 기반 메모 도구이다.
 사용자는 가사를 복사하여 붙여넣으면, 가사가 한 줄씩 오선지 위에 자동 배치된다.
 사용자는 음악 이론을 몰라도, 본인이 정의한 청음 기호를 드래그 앤 드롭으로 가사 위에 배치하여
 숨, 강조, 음의 높낮이, 길게 부르는 구간 등을 시각적으로 기록할 수 있다.
 
 핵심 가치는
 - 음악 용어 없이도 청음 기록 가능
 - 가사 + 멜로디 흐름을 한 화면에서 직관적으로 관리
 - 개인화된 청음 기호 체계 유지
 
 ---
 
 ## 2. 타깃 사용자
 
 - 보컬 연습자 (취미 / 지망생)
 - 음악 이론이 어려운 입문자
 - 보컬 트레이너 및 교육자
 - 작곡/편곡 초안 정리가 필요한 사용자
 
 ---
 
 ## 3. 사용자 문제 정의
 
 - 악보, 음악 이론 용어는 이해하기 어렵다
 - 청음 시 숨/강조/길게 부르는 포인트를 정리할 도구가 없다
 - 가사를 기반으로 멜로디 흐름을 시각화하고 싶다
 - 손으로 적는 청음 노트는 수정/저장이 불편하다
 
 ---
 
 ## 4. 핵심 사용자 시나리오
 
 1. 사용자는 노래 가사를 복사하여 앱에 붙여넣는다
 2. 가사는 자동으로 한 줄씩 오선지 라인에 배치된다
 3. 사용자는 청음 기호 도구 모음에서 기호를 선택한다
 4. 기호를 드래그하여 특정 단어/음절 위에 올려둔다
 5. 청음이 끝난 노트를 저장하거나 PDF/이미지로 내보낸다
 
 ---
 
 ## 5. 나만의 청음 기호 세트 (기본 제공)
 
 아래 기호들은 음악 용어가 아닌, 사용자의 직관을 기준으로 설계된다.
 
 기호 세트 정의
 
 - 기호: ✧
   - 의미: 숨 (필수)
   - 설명: 반드시 숨을 쉬어야 하는 지점
 - 기호: ○
   - 의미: 숨 (선택)
   - 설명: 상황에 따라 숨을 쉬어도 되는 지점
 - 기호: ★
   - 의미: 강조
   - 설명: 힘을 주거나 감정을 실어 부르는 음
 - 기호: ‼
   - 의미: 강한 강조
   - 설명: 클라이맥스, 가장 중요한 포인트
 - 기호: ↑
   - 의미: 음이 올라감
   - 설명: 이전 음보다 높아짐
 - 기호: ↓
   - 의미: 음이 내려감
   - 설명: 이전 음보다 낮아짐
 - 기호: ↑↑
   - 의미: 아주 높은 음
   - 설명: 고음 구간, 주의 필요
 - 기호: ↓↓
   - 의미: 아주 낮은 음
   - 설명: 저음 구간
 - 기호: ~~~
   - 의미: 길게 부르기
   - 설명: 음을 늘여서 유지
 - 기호: 🌀
   - 의미: 떨림
   - 설명: 비브라토, 흔들리는 소리
 - 기호: •
   - 의미: 작게
   - 설명: 힘을 빼고 가볍게 부름
 - 기호: →
   - 의미: 이어부르기
   - 설명: 끊지 않고 자연스럽게 연결
 
 ---
 
 ## 6. 커스텀 기호 기능
 
 - 사용자는 본인만의 기호를 추가할 수 있다
 - 기호는 텍스트, 이모지, 간단한 SVG 형태 허용
 - 기호 이름과 의미를 함께 저장
 - 즐겨찾기 기호 고정 가능
 
 ---
 
 ## 7. 오선지 기반 에디터 기능
 
 - 가사 붙여넣기 시 줄 단위로 자동 분리
 - 각 줄마다 오선지 라인 자동 생성
 - 오선지는 무한 스크롤 구조
 - 오선지 확대/축소(Zoom)
 - 전체 화면 드래그(Pan)
 
 ---
 
 ## 8. 가사 텍스트 처리
 
 - 가사는 단어 또는 음절 단위로 선택 가능
 - 기호는 특정 단어 위에 위치할 수 있음
 - 기호 위치는 가사에 상대적으로 고정됨
 - 가사 편집 시 기호 위치 자동 보정
 
 ---
 
 ## 9. 기호 배치 인터랙션
 
 - 기호 도구 모음에서 드래그 앤 드롭
 - 가사 단어 위에 자동 스냅
 - 기호 크기, 색상 변경 가능
 - 기호 복사 / 삭제 가능
 - 여러 기호를 그룹으로 이동 가능
 
 ---
 
 ## 10. 저장 및 내보내기
 
 - 자동 저장 (로컬 + 클라우드)
 - 수동 저장
 - PDF 내보내기
 - PNG 이미지 내보내기
 - JSON 형식으로 데이터 백업 및 복원
 
 ---
 
 ## 11. 데이터 구조 (개념)
 
 - Document
   - id
   - title
   - lyrics (line 배열)
   - symbols (기호 객체 배열)
   - createdAt
   - updatedAt
 - Symbol
   - symbolId
   - type (기본 / 커스텀)
   - position (lineIndex, wordIndex, offset)
   - style (color, size)
 
 ---
 
 ## 12. 비기능 요구사항
 
 - 데스크톱 우선 (태블릿 대응)
 - 마우스/트랙패드 드래그 최적화
 - 반응 속도 빠를 것 (기호 배치 지연 없음)
 - 학습 없이 바로 사용 가능해야 함
 
 ---
 
 ## 13. MVP 범위
 
 포함
 - 가사 붙여넣기
 - 오선지 자동 배치
 - 기본 청음 기호 세트
 - 기호 드래그 앤 드롭
 - 저장 / 이미지 출력
 
 제외 (차후)
 - 오디오 분석
 - 자동 음정 추출
 - 실시간 협업
 
 ---
 
 ## 14. 성공 지표 (KPI)
 
 - 첫 청음 노트 완성까지 소요 시간 5분 이내
 - 저장 기능 사용률 50% 이상
 - 재방문율 25% 이상
